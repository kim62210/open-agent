# Interaction Patterns

스킬 실행 중 사용자와의 상호작용이 필요한 경우의 패턴.

## 자동 진행 vs 사용자 확인 기준

| 상황 | 자동 진행 | 사용자 확인 필요 |
|------|----------|----------------|
| 읽기 전용 작업 (분석, 검색) | O | - |
| 새 파일 생성 (기존 파일 영향 없음) | O | - |
| 기존 파일 수정/덮어쓰기 | - | O |
| 외부 API 호출 (비용 발생 가능) | - | O |
| 여러 선택지 중 하나를 골라야 할 때 | - | O |
| 되돌리기 어려운 작업 | - | O |

## 확인 요청 패턴

SKILL.md body에서 사용자 확인이 필요한 시점을 명시:

```markdown
## 절차
1. 대상 파일 분석 (자동)
2. 변환 옵션 제시 — **사용자에게 아래 옵션을 안내하고 선택을 받을 것:**
   - A: 전체 변환 (원본 유지)
   - B: 선택적 변환 (변환할 항목 지정)
3. 선택된 옵션으로 변환 실행
```

## 진행 상황 보고 패턴

장시간 작업(3단계 이상 파이프라인)에서 중간 진행 보고:

```markdown
## 절차
1. 데이터 수집 → 완료 후 수집된 건수 보고
2. 데이터 처리 → 완료 후 처리 결과 요약 보고
3. 최종 출력 → 결과물 경로와 요약 제공
```

## 입력 요구 패턴

스킬이 외부 리소스(API 키, 파일 경로 등)를 필요로 할 때:

```markdown
## 사전 요구사항
이 스킬은 아래 정보가 필요합니다. 누락 시 사용자에게 요청할 것:
- **대상 파일 경로**: 변환할 파일의 절대 경로
- **출력 디렉토리**: 결과를 저장할 디렉토리 (미지정 시 원본과 같은 디렉토리)
```
