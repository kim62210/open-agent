# Error Handling Patterns

스킬에 스크립트나 외부 의존성이 있을 때 사용하는 에러 처리 패턴.

## 스크립트 실패 대응

스크립트가 실패할 수 있는 스킬은 SKILL.md body에 폴백 절차를 명시:

```markdown
## 절차
1. `run_skill_script("my-skill", "convert.py", [입력경로, 출력경로])` 실행
2. **실패 시**: 에러 메시지를 분석하고 아래 대응:
   - 파일 형식 오류 → 사용자에게 올바른 형식 안내
   - 의존성 누락 → 표준 라이브러리만 사용 가능 (pip 설치 불가). 대체 stdlib 모듈 사용 또는 스크립트 재설계
   - 타임아웃 → 입력 크기 줄여 재시도
3. **성공 시**: 출력 파일 존재 확인 후 다음 단계 진행
```

## 재시도 패턴

같은 스크립트를 인자를 바꿔 재시도할 수 있는 경우:

```markdown
## 변환 절차
1. 기본 옵션으로 실행: `run_skill_script("converter", "run.py", [파일, "--format", "auto"])`
2. 실패 시 명시적 포맷으로 재시도: `run_skill_script("converter", "run.py", [파일, "--format", "utf-8"])`
3. 재실패 시: 사용자에게 파일 인코딩 확인 요청
```

## 단계별 검증 패턴

파이프라인 스킬에서 중간 단계 실패 시 전체를 재시작하지 않는 패턴:

```markdown
## 실행 절차
1. 데이터 수집: `run_skill_script("pipeline", "step1_collect.py", [소스])`
   → 실패 시: 소스 접근 권한 확인, 네트워크 상태 확인
2. 데이터 처리: `run_skill_script("pipeline", "step2_process.py", [중간파일])`
   → 실패 시: step1 출력 형식 확인, step2만 재실행
3. 보고서 생성: `run_skill_script("pipeline", "step3_report.py", [처리결과])`
   → 실패 시: step2 출력으로 수동 요약 작성 (폴백)
```

## 가드레일 패턴

스킬이 하면 안 되는 작업을 명시하여 사고 방지:

```markdown
## 제약 사항
- **삭제 금지**: 원본 파일을 삭제하거나 덮어쓰지 말 것. 항상 새 파일로 출력
- **경로 제한**: 사용자가 지정한 디렉토리 외부에 파일을 생성하지 말 것
- **크기 제한**: 100MB 이상 파일은 처리하지 말고 사용자에게 분할 안내
```
